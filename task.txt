1. Создать приложение, состоящее из 2 docker контейнеров: frontend (html страница с формой отправки сообщения) и backend (принимает данные из формы и сохраняет сообщение в файле в приватном AWS S3 bucket).
 Язык программирования для backend произвольный.

2. В AWS создать приватный VPC:
- 2 public & 2 private subnets
- bastion host
- машины в приватной сети имеют доступ в интернет, но не имеют public IP, из интернета к машинам - только через bastion host по SSH

3. Развернуть приложение из п.1 в ECS кластере, размещенном в VPC из п.2:
- ECS на EC2
- доступ в приложение через AWS ALB
- не хранить креды для S3 в backend контейнере
- настроить auto-scaling по любой метрике (1 scale out, 1 scale in rule)

4. Создать Jenkins сервер на отдельной EC2 в этом же VPC. Написать Jenkins Pipeline для автоматического деплоя приложения (при появлении новых коммитов в master бранче) без даунтайма для приложения.
